# Query Execution and Interactive Answers

이 문서는 기존 README의 Part 3 질문 답변 고도화 기록과 현재 PANDAS 실행 구조를 정리합니다.

## QueryPlan 실행

표 질문은 제한된 JSON 계약인 QueryPlan으로 표현합니다.

지원 연산:

- `list`
- `count`
- `sum`
- `mean`
- `median`
- `mode`
- `min`
- `max`
- `group_sum`

지원 조건:

- 정확히 일치, 포함
- 초과, 이상, 미만, 이하
- 구간
- 빈 값, 값 존재
- 다중 조건의 AND/OR
- 정렬, 개수 제한, 순위, 동점 정책

```text
QueryPlan
→ DataFrame·컬럼 범위 검증
→ 자료형과 연산자 검증
→ 질문 원문 근거 검증
→ 결정적 실행
→ 답변과 구조화 근거 생성
```

LLM은 실행 계획을 제안할 수 있지만 DataFrame 코드를 직접 실행하지 않습니다.

## E1. 동일 이름과 동명이인

같은 표시 이름이 여러 행에 등장한다고 같은 사람으로 단정하지 않습니다.

- 연락처 등 원본 식별 근거가 같으면 같은 사람의 여러 결제 기록으로 연결
- 식별 근거가 다르면 동명이인 후보로 분리
- 마스킹 이름은 글자 수와 마스킹 위치가 일치할 때만 비교
- 서로 다른 실제 이름을 마스킹 패턴만으로 합치지 않음

개인별 누적 금액 순위와 개별 결제 행 순위도 구분합니다.

## E2. 출처와 계산 근거

PANDAS 답변에는 가능한 경우 다음 근거를 포함합니다.

- 사용 문서
- 적용 필터
- 날짜 기준 컬럼
- 계산 대상 컬럼
- 원본 행 수
- 조건 통과 행 수
- 유효·제외 행 수
- 고유 인원 수
- 정렬과 순위 정책

금액 컬럼이 여러 개면 질문과 명확히 연결되는 컬럼만 선택합니다. 구분할 수 없으면 계산하지 않고 후보 컬럼을 안내합니다.

## E3. 인물 카드와 금액 기록

API는 답변 문자열과 별도로 구조화 데이터를 반환합니다.

- 인물 entity
- 원본 속성
- 결제 기록
- 계산 상세 reference
- inline answer segments
- 페이지 정보

브라우저는 한국어 답변 문자열을 다시 파싱하지 않고 이 구조화 데이터를 사용해 이름과 금액을 클릭 가능한 요소로 표시합니다.

인물 카드 정책:

- 원본 데이터 컬럼만 표시
- OCR confidence와 내부 식별자 숨김
- 전화번호와 이메일은 일반 목록에서 숨김
- 사용자가 상세 카드를 열었을 때만 연락처 표시
- 동명이인은 후보 카드로 분리

계산 상세 정책:

- 계산에 기여한 원본 행 페이지 조회
- 한 페이지 기본 50건
- 전체 기여 행을 `list[dict]`로 미리 복제하지 않음
- 상세 저장소 TTL 15분
- 최대 256개, 오래된 항목부터 제거

## E4. 후속 질문 자동완성

현재 단계에서는 문서 스키마와 실제 인물 이름을 기준으로 다음 질문 후보를 제공합니다.

- 인물 금액
- 인물 전체 기록
- 원본 필드 조회
- 날짜 목록
- 날짜 합계
- 날짜 인원
- 기본 집계와 순위

후속 답변 문맥을 사용한 다단계 추천은 이후 확장 범위입니다.

## 목록 출력

일반 목록은 사용자가 읽기 쉽도록 이름 중심으로 출력합니다.

- 200명 이하: 전체 이름
- 200명 초과: 전체 건수와 처음 200명
- 세부 필드를 명시한 질문: 요청한 원본 컬럼 표
- 구조화 API 결과: 기본 50건

이 제한은 수만 건의 이름이 하나의 답변 문자열에 포함되어 서버와 브라우저가 느려지는 것을 막습니다.

| `query_plan.py` | 실행 계획 타입 계약 |

| `money.py` | 원, 천원, 만원 정규화 |
| `formatter.py` | 사용자 답변 |
| `interactive.py` | 인물·계산 상세 API 데이터 |
